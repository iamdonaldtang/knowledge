# API

API tasks rely on APIs provided by the project team to return whether a user has completed a specified task and to send the task completion data back to TaskOn.

For contract interaction tasks, configuring via smart contracts in the OnChain section of TaskOn’s Quest setup is the recommended approach.&#x20;

If using the API Task Module to set up tasks, please ensure proper configuration and submit the setup for our verification before proceeding.

TaskOn’s API verification task feature integrates APIs from various applications and functions. When using API verification tasks, provide the task name, description, task icon, and target URL.

There are two types of API tasks:

**Completion-Based API Task**:Focuses on task completion status, verified through an API interface.

**Performance-Based API Task:**&#x46;ocuses on dynamic point rewards. Project teams can set periodic reward actions via the API interface to specify which user actions qualify for ongoing rewards.

### Setting the API URL:

You can choose between social media accounts or wallet addresses as the basis for returning task completion verification results.
