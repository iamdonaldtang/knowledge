# On-Chain Data

### 8.1 Number of Transaction Addresses + Transaction Volume

#### Target

• Simultaneously ensuring trading volume and the number of participants. A tiered system provides a safety net at the bottom, while a ranking system incentivizes the top.

#### Recommended Tools

• Trading Race

#### Recommended Configuration

• Set up 3 reward pools

￮ Basic pool

▪ Top 10 in rewarded trading volume

▪ Reward: USDT (50% of budget)

￮ **Project Token Pool 1**

▪ Top 100 in rewarded trading volume

▪ Reward: Project Team Tokens (30% of the budget)

￮ **Project Token Pool 2**

▪ Rewards for transactions > 10U

▪ Reward: Project Team Tokens (20% of the budget)

### 8.2 Number of Transaction Addresses

#### Target

• Increase the number of addresses participating in Swap

#### Recommended Tools

• Task

#### Recommended Configuration

• **Task 1**

￮ Swap 1 U

￮ Reward 10 points

￮ Recurrence: once

**• Task 2**

￮ Swap 5 U

￮ Reward 40 points

￮ Egibility: Task 1 must be completed.

￮ Recurrence: once

**• Task 3**

￮ Swap 10 U

￮ Reward 70 points

￮ Egibility: Task 2 must be completed.

￮ Recurrence: Daily

#### Recommended Reward Methods

• Milestone (80% of budget)

• Leaderboard (20% of budget)

￮ Rewards for heads other than Milestone nodes
