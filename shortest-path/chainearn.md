## How to Create Your ChainEarn Task?

The OnChain task module directly targets on-chain interactions such as trading volume, cross-chain transaction volume, and the number of token-holding users. You can specify the on-chain actions you want users to complete and set targets for wallet addresses, trading volume, or transaction count. TaskOn will recommend these on-chain tasks to suitable users based on the pricing you set.

In the TaskOn backend controller, click _Onchain Boost_ to enter the ChainEarn task setup.

**Note**: ChainEarn is a task-matching feature. Based on your target budget and desired outcomes, we will create specific frontend tasks according to the tasks you submit. Your budget expenditure strictly adheres to your defined requirements and pricing. You can also negotiate to terminate the task at any time after publishing, and any unspent budget will be refunded to your wallet.

### **Hold Token**

Configure the Network where the _Hold Token_ resides, the Token Name, the minimum Token  Balance, and the minimum holding duration.

<figure><img src="https://lh7-rt.googleusercontent.com/docsz/AD_4nXeFGG2hiMlF6x2RQx9fn8aU7rof2QqD_kpDGYTv7RLbskiiGpW6cI3L7FtJj-b5A4v3_ukgnnkLxAaSndbfyWFkqA0nYPaJOEaoQlP2UvJ2O9nKMmDxtMgBZvYeweEW2iCmdxerDw?key=0Ugs8HwP_vZfQwQiqIZ9vA" alt=""><figcaption></figcaption></figure>

_Hold Token_ tasks can only select `Target Wallets` as the goal, as these tasks focus on reaching a specified number of unique wallet addresses to expand the user base.

In the _More about your request_ section, you can provide detailed notes to supplement your requirements and goals.

When setting up the task, it’s recommended to enable anti-bot measures to effectively filter out bot users.

Next, set the budget. The total budget amount directly impacts the final target wallet outcomes, so set it reasonably.

Finally, define the overall event schedule. Click _Next_.

<figure><img src="https://lh7-rt.googleusercontent.com/docsz/AD_4nXcGkBIg3sW5dtgzFzh_iTVwy7H8cPrhAoYlwdZD2-7x2b8DHO3hlYdpYrWBW0Pqe9FkJvvqKMXkodk8Vx1gI_l_mK5_zgQbq_dx6rul_fTqCfXKGdPbP3opZi3gzizLQYjAKj0c3g?key=0Ugs8HwP_vZfQwQiqIZ9vA" alt=""><figcaption></figcaption></figure>

After setting the target budget, deposit the budget into the TaskOn platform before submitting the task. TaskOn will create an execution plan that best meets your goals based on your budget.

### Swap

Configure the DEX, Network, _Swap from_ and _Swap to_ Tokens, and the minimum swap volume.

<figure><img src="https://lh7-rt.googleusercontent.com/docsz/AD_4nXd_Z_otp8cdYwosmELI1PzeGwO11l8ycdpe1MBtPpips8hy8X6d3U80j7-4KfO44vjqORLd4rf_G6ZIlpvHH7hcsIOpeCiMD6rSEmpyPTpwNNWMxG-4xNEzegLNKJ9luDup05sg8w?key=0Ugs8HwP_vZfQwQiqIZ9vA" alt=""><figcaption></figcaption></figure>

In the _Goals and Budget_ section, you can set the transaction target and total budget.

For swap transaction scenarios (especially for token project airdrops or incentive programs), you can choose your target from three key metrics: _Target Wallets_ (number of unique wallets), _Target Volumes_ (trading volume), and _Target Transactions_ (number of transactions). These incentivize goals across three dimensions: user reach, liquidity depth, and user activity.

* **Target Wallets**: Set the number of unique wallet addresses to reach, with the core goal of expanding the user base.
* **Target Volumes**: Set the total swap transaction volume target, e.g., incentivizing users to generate $1,000,000 in trading volume, with the core goal of increasing token liquidity depth.
* **Target Transactions**: Set the target number of valid swap transactions, which enhances Sybil resistance for the project ecosystem by encouraging frequent user interactions to boost on-chain activity and credibility.

Configure the reward amount, Network, Token, Schedule and total budget. Calculate the acquisition cost for each target value, then publish.
